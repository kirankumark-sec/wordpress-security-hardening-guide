# 🔐 Security Hardening Guides

A curated collection of **practical security hardening guides** for modern applications, frameworks, and platforms.  
This repository is designed to help developers, system administrators, and security teams implement **secure‑by‑default** configurations and reduce real‑world attack surface.

🌐 **Live Site (GitHub Pages):**  
https://kirankumark-sec.github.io/security-hardening-guide/

---

## 🎯 Purpose

The primary goal of this project is to:
- Provide **clear, actionable security best practices**
- Bridge the gap between **security recommendations and implementation**
- Support **secure deployment, VAPT remediation, and compliance efforts**
- Act as a **central reference** for application and platform security hardening

This project is intentionally kept **tool‑agnostic** and **practical**, focusing on what actually works in production environments.

---

## 📘 Available Guides

| Platform / Framework | Description |
|---------------------|-------------|
| **WordPress** | Secure configuration for WordPress core, plugins, themes, and admin access |
| **Node.js** | Application‑level and runtime security best practices |
| **React** | Frontend security recommendations to prevent common client‑side attacks |
| **Flutter** | Secure coding and deployment guidelines for Flutter mobile and web applications |

Each guide is provided as a **standalone HTML document**, making it easy to access, share, and host.

---

## 🧭 How to Use

1. Open the landing page:  
   **`index.html`** (or the GitHub Pages URL)
2. Select the required platform guide
3. Follow the listed hardening recommendations during:
   - Application development
   - Deployment
   - Security audits
   - Post‑VAPT remediation
